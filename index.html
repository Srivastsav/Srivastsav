CASE FILE 01

AI Portfolio Publisher & Website Updater v2

Manual portfolio updates are repetitive and time-consuming, involving documentation, GitHub uploads, and website edits.

  • Form submission captures structured project details.
  • A Code node prepares a controlled Gemini API request.
  • Gemini generates portfolio-ready content in structured JSON.
  • A parser extracts the AI output and keeps the original form data.
  • n8n generates README.md, setup-guide.md, workflow-summary.md, linkedin-post.md, and workflow.json files.
  • The project files are uploaded to the n8n portfolio GitHub repository.
  • The workflow retrieves the portfolio website index.html file from GitHub and decodes the base64 content.
  • A website project card is generated using AI-supported content.
  • The card is inserted or updated in the System Case Files section using project-specific HTML markers.
  • A new GitHub branch is created.
  • The updated index.html file is committed through the GitHub API.
  • A pull request is created so the website update can be reviewed and merged manually.

Significantly reduces manual effort, standardizes documentation, prevents duplicate website entries, and ensures a controlled review process for website updates.

Error handling is implemented for API calls and file operations. The GitHub pull request mechanism allows for manual review and correction of website updates before merging.
